# 规则管理 — 需求说明（含详情页、审核场景）

本文档描述「规则管理」模块的产品需求，包含 **规则列表页、规则详情（配置）页、审核场景配置** 三部分，供设计与研发对齐。  
**审核界面（ReviewerPage）已确认为准确基准**，本文档所有定义须与之保持一致。

---

## 1. 背景与目标

- 审核界面已确认并冻结；规则管理需补齐 **详情页 + 审核场景** 能力。
- 规则的核心能力是 **配置提示词**：分为 **系统提示词** 与 **用户提示词**。
- 规则配置后，通过 **审核场景** 绑定到具体产业公司/部门的 BPM 审核节点。

---

## 2. 风险维度（统一口径）

列表页与审核界面使用 **同一套风险分类**，不再使用「高/中/低」：

| 分类（category） | 列表标签 | 含义 | 颜色（已用） |
|------------------|----------|------|-------------|
| **trigger** | 触发风险 | 合同条款与规则要求不一致，触发判断规则 | `risk-high`（红） |
| **missing** | 信息缺失 | 合同中缺少支撑判断的原文依据 | `risk-medium`（琥珀） |
| **low** | 规则内 | 合同条款在规则要求范围内，低风险 | `risk-low`（绿） |

> **说明**：category 描述的是 **规则运行后** 对一份合同的判定结果类型。在规则配置时不需要选择 category——它是模型输出。规则列表/详情上应展示的是「规则类型」标签而非「风险等级」（见下方字段映射）。

---

## 3. 提示词模型

| 类型 | 维护方 | 说明 |
|------|--------|------|
| **系统提示词** | 开发侧约束；**超级管理员**可在界面编辑 | 每条规则 **独立一份**。典型内容：角色设定、输出 JSON 格式、安全边界、引用原文时必须携带 `locationId`（段落 ID）的要求等。 |
| **用户提示词** | **业务用户**编辑 | 描述本条规则具体审什么、判据、例外与业务口径等。支持 **变量**（变量清单见 §11）。 |

实际请求模型侧：**系统提示词 + 用户提示词 + 合同正文（含段落 ID）**。

---

## 4. 权限与页面形态

- **业务用户** 与 **超级管理员** 共用 **同一套规则详情页**。
- **系统提示词**：超级管理员可编辑；业务用户 **只读**。
- **用户提示词**：按业务角色策略可编辑。
- 通过权限控制同一页面上的编辑/只读状态，不拆多套页面。

---

## 5. 测试功能（保存非必须）

- 详情页提供 **规则测试**：
  - 携带当前 **系统提示词 + 用户提示词**；
  - **用户上传样本合同**（样本合同需经过解析生成段落 ID，与正式审核一致）；
  - 展示 **测试结果**，字段结构与审核界面一致（见 §8）；
  - 测试结果支持 **点击结果项跳转到样本合同对应段落**（与审核页「定位原文」同一套交互）。
- **保存规则不以测试为前置条件**：允许未测试即保存；可选非阻断提示。

---

## 6. 版本与历史

所有版本管理仅指 **规则版本**（模型版本暂不管理，视为单一版本）。

- **版本历史**：每次保存/发布生成新版本号，记录操作人、时间、变更内容摘要。
- **草稿**：未发布的编辑为草稿状态；发布后生效。
- **回滚**：可选择历史某一版本进行回滚，**回滚操作生成新版本记录**（可审计，不覆盖历史）。

---

## 7. 删除规则

- **仅当规则处于非生效状态**（如已停用）时才允许删除。
- 生效中的规则不可直接删除，需先停用。

---

## 8. 规则输出字段与审核界面的映射

审核界面每条结果（`RiskItem`）的字段已确认准确。规则配置与模型输出须保证能填充以下结构：

| 审核界面字段 | 类型 | 说明 | 规则配置侧对应 |
|-------------|------|------|---------------|
| `title` | string | 规则名称，如"交货方式要求" | 来自 **规则基础信息 → 名称** |
| `category` | `'trigger' \| 'missing' \| 'low'` | 判定结果类型 | **模型输出**（非规则配置字段） |
| `summary` | string | 一句话摘要，如"命中风险审核规则" | **模型输出**（基于规则判断生成） |
| `conclusion` | string | 审核结论 | **模型输出** |
| `reason` | string | 结论原因，含条款引用（如"根据第3条第3.1款…"） | **模型输出**（系统提示词约束格式） |
| `details` | string | 详细说明 | **模型输出** |
| `quote` | string? | 原文引用片段 | **模型输出**（系统提示词要求引用原文） |
| `locationId` | string? | 对应原文段落锚点，用于左侧跳转 | **模型输出**（系统提示词强制要求携带） |
| `reasonLocationId` | string? | 结论原因中引用的段落锚点 | **模型输出** |
| `structuredFields` | array | 评审要点提取 | 见下表 |

**structuredFields 子项**：

| 字段 | 说明 | 规则配置侧对应 |
|------|------|---------------|
| `label` | 要点名称，如"交货方式" | **建议**：规则详情可配置「需提取字段模板」（字段名列表），模型据此提取 |
| `value` | 提取值 | **模型输出** |
| `sourceText` | 原文出处 | **模型输出** |
| `locationId` | 段落锚点 | **模型输出** |
| `confirmed` | 是否已确认 | **前端状态**（审核员操作） |

### 规则详情页建议新增的配置项

基于上述映射，规则详情页除双提示词外，建议可配置：

1. **需提取字段模板**（structuredFields 的 label 列表）：如规则"交货方式要求"配置 `["交货方式", "交货地点"]`，模型按此列表提取。
2. **规则所属模块**（module）：与审核场景联动。

---

## 9. 段落 ID 与原文定位

- **段落 ID（locationId）在合同解析阶段生成**，随合同正文一起传入模型。
- **系统提示词必须要求模型**：在输出 `quote`、`reason`、`structuredFields` 中引用原文时，**必须携带对应的 locationId**。
- **审核界面** 依赖 locationId 实现左侧合同原文的高亮与滚动定位。
- **规则测试** 须与正式审核使用同一套解析 → 段落 ID 生成 → 模型调用 → 结果渲染流程，确保测试结果也支持段落跳转。

---

## 10. 审核场景配置（新增功能）

### 概念

**审核场景** = 针对特定产业公司/部门，选择多条规则组合为一个审核集合，对接到 BPM 系统的一个审核节点。

### 示例

| 场景名称 | 适用范围 | 包含规则 | 对接 BPM 节点 |
|----------|----------|----------|--------------|
| 财务部审核 | 低压电器公司 · 财务部 | R001 交货方式、R002 付款方式、R005 发票开具 | 财务部审核节点 |
| 法务合规审核 | 全集团 · 法务部 | R004 违约金上限、R001 交货方式 | 法务合规节点 |

### 功能要点

- **场景管理**：新建、编辑、停用/删除场景。
- **场景配置**：
  - 选择 **适用产业公司**（或全集团）；
  - 选择 **适用部门**；
  - 从已生效规则中 **勾选多条规则** 组成该场景的规则集；
  - 绑定 **BPM 审核节点标识**。
- **与审核界面的关系**：审核页头部已有「财务部视图」等提示（当前为 mock）；正式流程下，审核页根据 BPM 传入的场景 ID，加载该场景下的规则集进行评审，并在头部展示对应部门/场景名。
- **一条规则可被多个场景引用**（多对多）。
- **场景内规则排序**：可配置规则展示/执行顺序（影响审核页右侧卡片排列）。

### 与规则的状态联动

- 场景只能引用 **已生效** 规则；规则停用后，已引用该规则的场景需提示管理员处理。
- 场景自身也有 **生效/停用** 状态。

## 11. 待补充 / 实现时需细化

| 项 | 说明 |
|----|------|
| **用户提示词变量** | 第一版变量列表（如 `{{规则名}}`、`{{模块}}`、`{{合同类型}}` 等）、占位符语法、详情页是否提供「插入变量」面板。 |
| **规则状态机** | 建议：草稿 → 已生效 → 已停用；仅「已停用」可删。需与版本发布语义对齐。 |
| **回滚审计** | 回滚后版本号规则（如 v3 回滚到 v1 内容，生成 v4）、操作人记录。 |
| **BPM 对接协议** | 场景 ID、规则版本、审核结果 payload（含 ruleId + ruleVersion + category + structuredFields）的接口定义。 |
| **场景管理入口** | 建议在侧边栏增加独立菜单项（如「审核场景」），与「规则管理」并列。 |
